<objective>
## Goal
Add MongoDB persistence layer to store processed document trees with versioning and metadata querying capabilities.

## Purpose
Enable production workloads by persisting processed documents for reuse, avoiding costly re-processing, and supporting temporal queries through document versioning.

## Output
- MongoDB infrastructure via docker-compose
- Persistence module with document versioning
- CLI integration with `--persist` flag
</objective>

<acceptance_criteria>

## AC-1: MongoDB Infrastructure
```gherkin
Given a clean development environment
When docker-compose up -d is executed
Then MongoDB is accessible at mongodb://localhost:27017
And the database "pageindex" is created on first use
```

## AC-2: Document Persistence
```gherkin
Given a processed document tree with metadata
When the persist flag is enabled and save is called
Then the document is stored in MongoDB with a unique document_id
And metadata fields are queryable (filename, upload_date, page_count, etc.)
And the version number is incremented for reprocessed documents
And is_latest flag is true only for the most recent version
```

## AC-3: CLI Integration
```gherkin
Given the PageIndex CLI
When --persist flag is provided with a document path
Then the processed tree is saved to MongoDB (not just file)
When --persist flag is omitted
Then the existing file-based output behavior is unchanged
```

## AC-4: Document Retrieval by Metadata
```gherkin
Given documents stored in MongoDB
When querying by filename or document_id
Then the latest version (is_latest=true) is returned
When querying by date range or tags
Then matching documents are returned
```

<task type="auto">
  <name>Task 2: Create persistence module</name>
  <files>pageindex/persistence.py, pageindex/__init__.py</files>
  <action>
    Create the persistence module with versioning support:

    1. Create `pageindex/persistence.py` with:
       - `DocumentVersion` dataclass matching the schema in CONTEXT.md
       - `PageIndexRepository` class with methods:
         - `save(document_id, tree, metadata)` → creates new version, updates is_latest
         - `get_latest(document_id)` → returns most recent version
         - `get_version(document_id, version_num)` → returns specific version
         - `find_by_metadata(filters)` → query by filename, tags, date range
         - `list_versions(document_id)` → returns version history
       - `get_mongo_client()` → connection factory from env var MONGODB_URI
       - `generate_document_id()` → UUID-based ID generation

    2. Update `pageindex/__init__.py`:
       - Export `PageIndexRepository`, `DocumentVersion`

    Schema implementation:
    ```python
    {
        "document_id": str,      # UUID, consistent across versions
        "version": int,          # Auto-incremented
        "created_at": datetime,  # ISO-8601
        "is_latest": bool,       # True for most recent only
        "tree": dict,            # The processed tree structure
        "metadata": {
            "filename": str,
            "file_path": str,
            "upload_date": datetime,
            "page_count": int,
            "token_count": int,
            "model_used": str,
            "processing_version": str,
            "tags": list[str],
            "doc_type": str
        }
    }
    ```

    Avoid: Using raw dict operations - use typed dataclasses. Don't implement query server - just storage module.
  </action>
  <verify>python -c "from pageindex import PageIndexRepository; print('OK')"</verify>
  <done>AC-2 and AC-4 satisfied: Persistence module with versioning and querying</done>
</task>

<task type="auto">
  <name>Task 3: Integrate persistence into CLI</name>
  <files>run_pageindex.py</files>
  <action>
    Add persistence option to the CLI:

    1. Add new arguments to `run_pageindex.py`:
       - `--persist` / `--no-persist` (default: no-persist) - store in MongoDB
       - `--document-id` (optional) - specify existing document_id for versioning
       - `--tags` (optional) - comma-separated tags
       - `--doc-type` (optional) - document category

    2. After processing completes:
       - If `--persist` is set:
         - Extract metadata from processed document
         - Generate or use provided document_id
         - Save via PageIndexRepository
         - Print document_id and version number
       - If `--persist` is not set:
         - Keep existing file-based output behavior unchanged

    3. Handle MongoDB connection:
       - Read MONGODB_URI from environment (default: mongodb://localhost:27017)
       - Graceful error if MongoDB unreachable and --persist requested

    Avoid: Breaking existing file output - it should still work. Don't require MongoDB for normal operation.
  </action>
  <verify>python run_pageindex.py --pdf_path tests/pdfs/sample.pdf --persist && echo "Persistence works"</verify>
  <done>AC-3 satisfied: CLI --persist flag stores to MongoDB</done>
</task>
